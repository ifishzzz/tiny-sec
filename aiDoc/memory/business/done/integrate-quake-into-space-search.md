# 整合 Quake 与网络空间搜索

## 基本信息

- 提出日期：2026-05-12
- 当前状态：`done`
- 需求类型：前后端功能扩展
- 优先级：高

## 用户原始意图摘要

在已有 FOFA 搜索基础上，整合 Quake 搜索，并把菜单名称统一改成“网络空间搜索”。

## 影响范围

- 后端：统一搜索接口支持 `FOFA` / `Quake`
- 前端：同一页面内切换搜索引擎并复用结果表格
- 数据：补充 `quake_key` 参数位，更新现网菜单与 API 描述
- 文档：更新业务需求记忆与索引

## 涉及对象

- 接口：`/fofa/search`
- 页面：`view/fofa/index.vue`
- 配置：系统参数表中的 `fofa_email`、`fofa_key`、`quake_key`
- 菜单：`FOFA` 菜单更名为“网络空间搜索”

## 已确认约束

- 继续沿用非插件实现
- FOFA 与 Quake 都通过后端代理请求
- 两种引擎共用一个页面，不拆成两个独立菜单
- 现有路由权限继续沿用 `GET /fofa/search`

## 当前进展

- 已为统一搜索接口增加 `engine` 参数，支持 `fofa` 和 `quake`
- 已新增 Quake 查询逻辑，并从系统参数表读取 `quake_key`
- 已把前端页面改成统一的“网络空间搜索”页
- 已在页面中增加当前引擎对应的搜索规则、常用字段与示例提示
- 已同步更新当前数据库中的菜单标题、API 分组与 `quake_key` 参数占位

## 后续待办

- 在参数管理里填写真实的 `quake_key`
- 需要时可继续补充更多 Quake 结果字段展示
